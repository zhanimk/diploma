4. Пользовательские роли
В системе предусмотрено четыре типа пользователей.

4.1 Администратор
Администратор управляет системой соревнований.
Функции:
создание турниров


публикация положения турнира


управление категориями


просмотр заявок


подтверждение или отклонение участников


формирование турнирных сеток


управление расписанием


управление татами


просмотр результатов



4.2 Тренер
Функции тренера:
регистрация в системе


подтверждение спортсменов


загрузка официальной заявки клуба


просмотр турниров


управление участием своих спортсменов



4.3 Спортсмен
Функции спортсмена:
регистрация аккаунта


указание тренера


просмотр турниров


подача заявки на участие


просмотр турнирной сетки


просмотр истории выступлений



4.4 Судья
Судья управляет поединками.
Функции:
вход в систему судей


просмотр назначенных боёв


управление табло поединка


выставление оценок


подтверждение победителя



5. Регистрация пользователей
При регистрации пользователь указывает:
ФИО


номер телефона


дату рождения


клуб


город


роль пользователя


После регистрации выполняется:
SMS-подтверждение номера телефона.

6. Создание турнира
Администратор создаёт турнир.
Параметры турнира:
название


дата проведения


место проведения


возрастные категории


весовые категории


пол


количество татами


дедлайн регистрации


положение турнира (PDF)


После публикации турнир становится доступен пользователям.

7. Подача заявки на участие
Спортсмен:
открывает турнир


читает положение


подаёт заявку


Спортсмен должен быть прикреплён к тренеру.

8. Подтверждение тренером
После подачи заявки:
тренер получает уведомление


тренер подтверждает спортсмена


Без подтверждения тренера заявка не принимается.

9. Подтверждение администратором
Администратор проверяет:
данные спортсмена


подтверждение тренера


заявку клуба


После этого администратор:
принимает заявку
 или


отклоняет её



10. Закрытие регистрации
После наступления дедлайна:
регистрация автоматически закрывается


формируется окончательный список участников



11. Формирование турнирных сеток
После закрытия регистрации система:
группирует спортсменов по категориям


формирует турнирные сетки


Сетка создаётся по:
возрасту


весу


полу


Система автоматически создаёт:
раунды


список боёв


порядок проведения боёв



12. Табло поединка (система судейства)
В системе реализуется цифровое табло боя.
Судья управляет:
таймером поединка


оценками спортсменов


Возможные оценки:
Ippon — мгновенная победа


Waza-ari


Golden Score



13. Автоматическое обновление результатов
После завершения боя система автоматически:
фиксирует победителя


обновляет турнирную сетку


переводит победителя в следующий раунд


Результаты автоматически отображаются:
в турнирной сетке


в профиле спортсмена


в статистике турнира



14. Автоматическое распределение боёв
Система распределяет поединки:
по татами


по времени


Система рассчитывает:
примерное время начала боя


порядок выхода спортсменов



15. История спортсмена
В профиле спортсмена хранится:
список турниров


результаты боёв


занятые места


статистика побед
